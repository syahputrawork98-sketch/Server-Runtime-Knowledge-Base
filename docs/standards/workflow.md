# Standard Workflow (PPM V4)

Proses penulisan materi mengikuti 4 tahapan (PPM):

1. **Phase 1: Research & Accuracy**: Merujuk pada dokumentasi resmi Node.js dan source code Libuv.
2. **Phase 2: Decomposition**: Memecah materi menjadi unit kecil (CH/SEC).
3. **Phase 3: Implementation**: Menulis README dan Lab (contoh kode).
4. **Phase 4: Verification**: Memastikan diagram Mermaid akurat dan kode lab berjalan.

## "Gold Standard" Unit Delivery
Setiap unit harus menjawab:
- **What**: Apa itu?
- **Why**: Mengapa ini penting bagi skalabilitas server?
- **How**: Bagaimana mekanismenya di level OS (epoll/kqueue)?
- **Pitfalls**: Bahaya CPU-blocking di main thread.
